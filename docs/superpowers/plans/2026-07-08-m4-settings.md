# M4 Settings UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the three M4 Settings features: a **keybindings editor** with a capture flow, conflict detection, and live rebinding (no restart); **agent management** (a default agent for the launcher plus per-agent extra args and env overrides composed into every spawn — the local-LLM enabler); and **themes** (JSON token files under `userData/themes/`, live CSS-variable + xterm application, shipped presets, a folder button, and malformed-file fallback with a visible notice). Provider tokens are permanently dropped and must not appear anywhere.

**Architecture:** The real engineering is live propagation, not the forms. Keybindings are loaded once in main and consumed by three sinks (renderer dispatcher, main's webview key-forwarder, Settings display); a new `keybindings:set`/`reset` IPC validates + writes the file + updates main's in-memory copy + pushes the full map on `keybindings:changed`. Main's `keybindings` becomes a mutable `let`; `installWebviewPolicy` returns an `updateBindings` handle (its `before-input-event` closure reads a mutable `parsed` array) so forwarding follows without re-installing; the renderer dispatcher's mount-once effect re-parses into its **in-place** `bindings` array on every `keybindings:changed`. Agent management and themes both persist to `config.json`, which is owned by `AgentRegistry` (single writer, unknown-key `extra` preservation) — so `defaultAgent`, `agents`, and `theme` become **known typed keys** there, and the spawn path gains optional `extraArgs`/`env` composed in `SessionManager.spawn`. Themes are a shared token module (`src/shared/theme.ts`) applied by the renderer as `:root` CSS variables + xterm `ITheme`, seeded as preset JSON files on startup, resolved with a default-fallback that carries a visible error notice.

**Tech Stack:** Electron main/preload/renderer, React 19, xterm, Tailwind v4 (`@theme` tokens are real runtime-overridable `:root` custom properties), vitest (`tests/unit/**`, node env), Playwright `_electron` e2e (`launchApp`, `SAIIFE_E2E`).

Spec: `docs/superpowers/specs/2026-07-08-m4-settings-design.md` (approved 2026-07-08).

## Global Constraints

- Conventional Commits, subject ≤ 50 chars, body lines ≤ 100 chars (husky + CI commitlint; PR titles ≤ 50 chars); `npm run check` (eslint + prettier + typecheck + vitest) green before every commit.
- **Provider tokens are dropped permanently** (spec header): no safeStorage, no secret fields, no provider-token UI or IPC anywhere in this plan. Local-LLM support arrives only through agent env/args overrides.
- Config-as-code round-trips: GUI edits and hand edits of `config.json` and `keybindings.json` must both survive. Hand edits win on the next load. `config.json` is written by exactly one writer, `AgentRegistry.saveAgentConfig`.
- **`config.json` schema (this milestone adds three KNOWN top-level keys):** `defaultAgent: <AgentId>`, `agents: { "<agentId>": { "extraArgs"?: string, "env"?: { "KEY": "VALUE" } } }`, `theme: "<name>"`. These become typed fields of `AgentConfig` and are added to `KNOWN_TOP_LEVEL_KEYS` — saiife both reads AND writes them through the GUI, unlike the read-only hand-edited `environments` names map, which stays on `extra`. `environments` MUST keep riding `extra` (do not add it to `KNOWN_TOP_LEVEL_KEYS`).
- **Keybinding grammar (unchanged):** `parseBinding` requires ≥1 modifier + a key (`cmd+shift+x`); digits match by physical key (`e.code === 'Digit<N>'`) as well as `e.key`. Capture serializes a `KeyboardEvent` to that grammar in canonical modifier order `cmd, ctrl, alt, shift`; digits serialize from `e.code`. A modifier-only or unmodified press serializes to `null` (capture keeps waiting). Escape (no modifiers) cancels capture.
- **Live keybindings, no restart:** `keybindings:set(action, binding)` / `keybindings:reset(action)` / `keybindings:resetAll()` in main validate → write file → reassign the in-memory `let keybindings` → `webviewPolicy.updateBindings(next)` → `sendToWindow('keybindings:changed', next)`. The renderer dispatcher re-parses the pushed map into its existing `bindings` array **in place** (`bindings.length = 0; bindings.push(...)`) so the mount-once `onKey` closure sees it.
- **Themes path & fallback:** a theme is `userData/themes/<name>.json`. `config.json` `theme` absent ⇒ built-in dark default. A missing/malformed/unparseable theme file falls back to the default theme and surfaces a visible notice — never a crash. Shipped presets (seeded on startup if absent): `dark` (current default), `light`, `solarized-dark`, `nord`.
- **DOM/e2e contracts (new):** keybindings rows `.kb-row[data-action]` with `.kb-capture` (the clickable current-binding / "press keys…" control), `.kb-reset`, and `.kb-reset-all`; agent cards `.agent-card[data-agent]` with `.agent-default[data-agent]`, `.agent-args`, `.agent-env`; themes `.theme-select`, `.theme-open-folder`, `.theme-notice`; unchanged: `.landing select`, `.new-session`, `.pane[data-pane-id][data-status]`, `.session-row`. The theme applier writes `--color-surface` (and its siblings) on `document.documentElement`, the e2e probe.
- Button mousedown discipline: every new button gets `onMouseDown={(e) => e.preventDefault()}`; the keybinding-capture control and the env/args inputs are the documented exceptions that must take focus.
- **Capture vs dispatcher:** while a keybinding capture is active the editor sets `document.documentElement.dataset.capturingKeybind = '1'`; the renderer dispatcher's `onKey` returns immediately when that flag is set, so a captured combo that happens to already be bound never fires its action mid-capture.
- Out of scope (spec §"Out of scope"): provider tokens (dropped), local-LLM recipe docs, terminal-convenience actions, layout/density preferences, theme marketplace/import UI beyond the folder button.

```ts
const MODIFIER_KEYS = new Set(['Meta', 'Control', 'Alt', 'Shift'])

// Inverse of NAMED_KEY_MAP: KeyboardEvent.key/'code' spelling -> binding token.
const NAMED_KEY_REVERSE: Record<string, string> = {
  Enter: 'enter',
  Escape: 'escape',
  Tab: 'tab',
  ' ': 'space',
  ArrowLeft: 'arrow-left',
  ArrowRight: 'arrow-right',
  ArrowUp: 'arrow-up',
  ArrowDown: 'arrow-down'
}

/**
 * Turns a captured KeyboardEvent into the binding grammar (`cmd+shift+x`),
 * or null when the press is not a committable binding yet: a bare modifier
 * (still waiting for the real key) or a key with no modifier (bindings
 * require at least one, matching parseBinding). Modifiers are emitted in the
 * canonical order cmd, ctrl, alt, shift. Digits come from e.code so a
 * shift-mangled e.key ('!' for '1') still serializes to the physical digit.
 */
export function serializeKeyEvent(e: KeyEventLike): string | null {
  if (MODIFIER_KEYS.has(e.key)) return null
  const mods: string[] = []
  if (e.metaKey) mods.push('cmd')
  if (e.ctrlKey) mods.push('ctrl')
  if (e.altKey) mods.push('alt')
  if (e.shiftKey) mods.push('shift')
  if (mods.length === 0) return null

  const digit = e.code ? /^Digit([0-9])$/.exec(e.code) : null
  let key: string
  if (digit) {
    key = digit[1]
  } else if (NAMED_KEY_REVERSE[e.key]) {
    key = NAMED_KEY_REVERSE[e.key]
  } else if (e.code && /^Key([A-Z])$/.test(e.code)) {
    key = e.code.slice(3).toLowerCase()
  } else if (e.key.length === 1) {
    key = e.key.toLowerCase()
  } else {
    return null
  }
  return [...mods, key].join('+')
}

/**
 * Other actions whose current binding parses to the same combo as `binding`
 * would for `action`. The editor shows these instead of silently letting two
 * actions collide. Returns [] when the candidate is unparseable (nothing to
 * conflict with yet).
 */
export function findConflicts(
  bindings: Record<KeyAction, string>,
  action: KeyAction,
  binding: string
): KeyAction[] {
  const target = parseBinding(binding)
  if (!target) return []
  return bindingEntries(bindings)
    .filter(([a, b]) => {
      if (a === action) return false
      const p = parseBinding(b)
      return (
        p != null &&
        p.cmd === target.cmd &&
        p.ctrl === target.ctrl &&
        p.alt === target.alt &&
        p.shift === target.shift &&
        p.key.toLowerCase() === target.key.toLowerCase()
      )
    })
    .map(([a]) => a)
}
```

```ts
export function installWebviewPolicy(opts: {
  bindings: Record<KeyAction, string>
  onAction: (action: KeyAction) => void
}): { updateBindings(bindings: Record<KeyAction, string>): void } {
  session.fromPartition(BROWSER_PARTITION).setPermissionRequestHandler((_wc, _permission, cb) => {
    cb(false)
  })

  // Mutable: rebinding replaces the compiled list in place, so the
  // before-input-event closure below follows without re-installing the
  // web-contents-created handler.
  const compile = (bindings: Record<KeyAction, string>): [KeyAction, ParsedBinding][] =>
    bindingEntries(bindings).flatMap(([action, binding]) => {
      const p = parseBinding(binding)
      return p ? ([[action, p]] as [KeyAction, ParsedBinding]) : []
    })
  let parsed = compile(opts.bindings)

  app.on('web-contents-created', (_event, contents) => {
    if (contents.getType() !== 'webview') return

    contents.setWindowOpenHandler(({ url }) => {
      if (isHttpUrl(url)) void shell.openExternal(url)
      return { action: 'deny' }
    })
    contents.on('will-navigate', (event, url) => {
      if (!isHttpUrl(url)) event.preventDefault()
    })
    contents.on('will-frame-navigate', (event) => {
      if (!isHttpUrl(event.url)) event.preventDefault()
    })
    contents.on('will-redirect', (event) => {
      if (!isHttpUrl(event.url)) event.preventDefault()
    })
    contents.on('before-input-event', (event, input) => {
      if (input.type !== 'keyDown' && input.type !== 'rawKeyDown') return
      const like: KeyEventLike = {
        key: input.key,
        metaKey: input.meta,
        ctrlKey: input.control,
        altKey: input.alt,
        shiftKey: input.shift,
        code: input.code
      }
      const match = parsed.find(([, binding]) => eventMatches(binding, like))
      if (!match) return
      event.preventDefault()
      opts.onAction(match[0])
    })
  })

  return {
    updateBindings(bindings: Record<KeyAction, string>): void {
      parsed = compile(bindings)
    }
  }
}
```

```ts
  // webview key-forwarder, and push the full map so the renderer dispatcher
  // re-parses. No restart.

      [action as KeyAction]: DEFAULT_BINDINGS[action as KeyAction]
    })
  })
  ipcMain.handle('keybindings:resetAll', () => applyKeybindings({ ...DEFAULT_BINDINGS }))
```

```tsx
  const [capturing, setCapturing] = useState<KeyAction | null>(null)

    null

  useEffect(() => {
    let cancelled = false
    void window.saiife.getKeybindings().then((b) => {
      if (!cancelled) setBindings(b)
    })
    const off = window.saiife.onKeybindingsChanged((b) => setBindings(b))
    return () => {
      cancelled = true
      off()
    }
  }, [])

  // Capture: a window-level capture-phase listener owns the keyboard while a
  // row is armed. The dataset flag makes App's dispatcher stand down so an
  // already-bound combo cannot fire mid-capture. Escape (no mods) cancels.
  useEffect(() => {
    if (!capturing) return
    document.documentElement.dataset.capturingKeybind = '1'
    const handler = (e: KeyboardEvent): void => {
      if (e.key === 'Escape' && !e.metaKey && !e.ctrlKey && !e.altKey) {
        e.preventDefault()
        e.stopPropagation()
        setCapturing(null)
        return
      }
      const combo = serializeKeyEvent(e)
      if (combo === null) return // modifier-only / unmodified: keep waiting
      e.preventDefault()
      e.stopPropagation()
      const taken = findConflicts(bindings, capturing, combo)
      if (taken.length > 0) {
        setConflict({ action: capturing, combo, taken })
        setCapturing(null)
        return
      }
      const action = capturing
      setCapturing(null)
      setConflict(null)
      void window.saiife.setKeybinding(action, combo).then((next) => {
        if (next) setBindings(next)
      })
    }
    window.addEventListener('keydown', handler, true)
    return () => {
      delete document.documentElement.dataset.capturingKeybind
      window.removeEventListener('keydown', handler, true)
    }
  }, [capturing, bindings])

  const resetAll = (): void => {
    setConflict(null)
    void window.saiife.resetAllKeybindings().then(setBindings)
  }
  const resetOne = (action: KeyAction): void => {
    setConflict((c) => (c?.action === action ? null : c))
    void window.saiife.resetKeybinding(action).then(setBindings)
  }

  return (
    <div className="flex flex-col gap-4">
      <div className="flex items-center justify-between">
        <p className="m-0 text-[13px] text-gray-500">
          Click a shortcut, then press the new combination. Escape cancels. Edits apply

                reset
              </button>
            </div>
          ))}
        </div>
      ))}
    </div>
  )
}
```

```ts
/**
 * Splits a CLI-argument string into argv, honoring single/double quotes so
 * `--prompt "a b"` stays one arg. Used to append per-agent extra args at
 * spawn (the local-LLM enabler). Minimal on purpose — no backslash escapes,
 * no env expansion; the shell that already ran the agent handles the rest.
 */
export function splitArgs(input: string): string[] {
  const out: string[] = []
  let cur = ''
  let quote: '"' | "'" | null = null
  let has = false
  for (const ch of input) {
    if (quote) {
      if (ch === quote) quote = null
      else cur += ch
    } else if (ch === '"' || ch === "'") {
      quote = ch
      has = true
    } else if (/\s/.test(ch)) {
      if (has) {
        out.push(cur)
        cur = ''
        has = false
      }
    } else {
      cur += ch
      has = true
    }
  }
  if (has) out.push(cur)
  return out
}
```

```ts
const KNOWN_AGENT_IDS: AgentId[] = ['claude', 'codex', 'gemini', 'custom']

function parseAgentOverride(raw: unknown): AgentOverride | null {
  if (typeof raw !== 'object' || raw === null || Array.isArray(raw)) return null
  const out: AgentOverride = {}
  const extraArgs = (raw as { extraArgs?: unknown }).extraArgs
  if (typeof extraArgs === 'string' && extraArgs.trim().length > 0) out.extraArgs = extraArgs
  const env = (raw as { env?: unknown }).env
  if (typeof env === 'object' && env !== null && !Array.isArray(env)) {
    const cleaned: Record<string, string> = {}
    for (const [k, v] of Object.entries(env)) {
      if (k.length > 0 && typeof v === 'string') cleaned[k] = v
    }
    if (Object.keys(cleaned).length > 0) out.env = cleaned
  }
  return out.extraArgs || out.env ? out : null
}

function parseAgents(raw: unknown): AgentConfig['agents'] | undefined {
  if (typeof raw !== 'object' || raw === null || Array.isArray(raw)) return undefined
  const out: Partial<Record<AgentId, AgentOverride>> = {}
  for (const id of KNOWN_AGENT_IDS) {
    const override = parseAgentOverride((raw as Record<string, unknown>)[id])
    if (override) out[id] = override
  }
  return Object.keys(out).length > 0 ? out : undefined
}
```

```ts
/**
 * Validates a hand-written or shipped theme structurally (colors are any
 * non-empty string — CSS accepts names and hex alike). Returns null on any
 * shape violation; callers fall back to the default with a visible notice.
 */
export function parseTheme(raw: unknown): Theme | null {
  if (typeof raw !== 'object' || raw === null || Array.isArray(raw)) return null
  const t = raw as Record<string, unknown>
  if (!isStr(t.name)) return null
  const app = t.app as Record<string, unknown> | undefined
  if (typeof app !== 'object' || app === null) return null
  const appKeys = [
    'surface', 'surfaceRaised', 'sidebar', 'text',
    'working', 'needsYou', 'idle', 'running', 'exited'
  ] as const
  for (const k of appKeys) if (!isStr(app[k])) return null
  const term = t.terminal as Record<string, unknown> | undefined
  if (typeof term !== 'object' || term === null) return null
  for (const k of ['background', 'foreground', 'cursor', 'selectionBackground'] as const) {
    if (!isStr(term[k])) return null
  }
  const ansi = term.ansi
  if (!Array.isArray(ansi) || ansi.length !== 16 || !ansi.every(isStr)) return null
  const font = t.font as Record<string, unknown> | undefined
  if (typeof font !== 'object' || font === null) return null
  if (!isStr(font.family) || typeof font.size !== 'number' || !Number.isFinite(font.size)) {
    return null
  }
  return {
    name: t.name as string,
    app: {
      surface: app.surface as string,
      surfaceRaised: app.surfaceRaised as string,
      sidebar: app.sidebar as string,
      text: app.text as string,
      working: app.working as string,
      needsYou: app.needsYou as string,
      idle: app.idle as string,
      running: app.running as string,
      exited: app.exited as string
    },
    terminal: {
      background: term.background as string,
      foreground: term.foreground as string,
      cursor: term.cursor as string,
      selectionBackground: term.selectionBackground as string,
      ansi: ansi as string[]
    },
    font: { family: font.family as string, size: font.size as number }
  }
}
```

```ts
import { existsSync, mkdirSync, readdirSync, readFileSync, writeFileSync } from 'node:fs'
import { join } from 'node:path'
import { DEFAULT_THEME, PRESET_THEMES, parseTheme, type Theme } from '../shared/theme'

/** Writes each shipped preset into the themes dir if it is not already there. */
export function ensureThemesSeeded(dir: string): void {
  mkdirSync(dir, { recursive: true })
  for (const preset of PRESET_THEMES) {
    const file = join(dir, `${preset.name}.json`)
    if (!existsSync(file)) writeFileSync(file, JSON.stringify(preset, null, 2))
  }
}

/** Theme names available on disk (file basenames), sorted. */
export function listThemeNames(dir: string): string[] {
  try {
    return readdirSync(dir)
      .filter((f) => f.endsWith('.json'))
      .map((f) => f.slice(0, -'.json'.length))
      .sort()
  } catch {
    return PRESET_THEMES.map((t) => t.name)
  }
}

/**
 * Resolves the configured theme. A null name (config `theme` absent) is the
 * built-in dark default. A named file that is missing/malformed falls back to
 * the default and carries a human notice — never throws, never crashes.
 */
export function resolveTheme(
  dir: string,
  name: string | null
): { name: string; theme: Theme; error?: string } {
  if (name === null) return { name: DEFAULT_THEME.name, theme: DEFAULT_THEME }
  try {
    const parsed = parseTheme(JSON.parse(readFileSync(join(dir, `${name}.json`), 'utf8')))
    if (parsed) return { name, theme: parsed }
  } catch {
    /* fall through to the default + notice */
  }
  return {
    name: DEFAULT_THEME.name,
    theme: DEFAULT_THEME,
    error: `Theme “${name}” is missing or malformed — using the default.`
  }
}
```

**Known traps designed around (verified against current code):**
- Dispatcher parses bindings once in a mount-once effect (App.tsx line ~213): Task 1 refills the same `bindings` array in place (`bindings.length = 0; push`) via `onKeybindingsChanged` inside that same effect, so the stable `onKey` closure sees new bindings without re-subscribing the keydown listener.
- webview-policy received `bindings` by value at install: Task 1 refactors it to compile into a mutable `let parsed` and return `updateBindings`, which main calls on every change — forwarding follows rebinds with no re-install.
- config.json unknown-key `extra` preservation: `defaultAgent`/`agents`/`theme` are made KNOWN typed keys (added to `KNOWN_TOP_LEVEL_KEYS`) because saiife reads AND writes them via the GUI; `environments` names stay on `extra` (read-only). A regression test asserts `environments` still round-trips on `extra` alongside the new keys, and that `setDefaultAgent`/`setAgentOverride`/`setTheme` preserve it.
- Settings fetched agents on mount only: every override/default/theme mutation re-lists (`setAgents(updated)` / re-fetch), and Landing re-mounts (and re-fetches) when navigating back to Overview, so preselection reflects the new default.
- Capture vs global dispatcher (both capture-phase): the editor sets `document.documentElement.dataset.capturingKeybind` and the dispatcher bails on it, so capturing a currently-bound combo can't fire that action mid-capture.

**Risks / accepted trade-offs:**
- The capture effect depends on `bindings` (to compute conflicts), so it re-subscribes its window listener when the local bindings state updates — harmless (the guard/flag is idempotent) and only while a row is armed.
- `defaultValue` (uncontrolled) on the args/env fields commits on blur, not per keystroke — chosen to avoid re-render churn; a re-list after save keeps the sibling field's source-of-truth current.
- Env override precedence is `process.env < injection.env < user override`; documented that hook vars are saiife-internal so real collisions don't occur — if a user deliberately shadows a hook var they accept a degraded status feed.
- The e2e cannot assert the webview-forwarded rebind path (needs a live browser pane + focus); that path is covered by Task 1's unit-level `updateBindings` wiring and typecheck. The dispatcher rebind IS exercised end-to-end (cmd+y toggles the sidebar live).
- Theme presets ship fixed 16-color ansi palettes (published Solarized Dark / Nord values); users extend via the folder. Layout/density tokens are explicitly out of scope (spec §3).
