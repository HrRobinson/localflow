# M2.5 Needs-You Quick Actions Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let the user act on needs-you (yellow) sessions without entering the terminal: `cmd+u` jumps to / cycles through waiting panes, and an Approve control on pane headers and overview rows sends Enter to the session's pty — always showing a peek of the pending question first, never blind.

**Architecture:** Main process grows a per-session output peek (last N cleaned lines derived from the existing raw `tail` buffer, exposed over a new `session:peek` IPC invoke). The renderer gets one new keybinding action (`focus-needs-you`, default `cmd+u`) dispatched through the existing capture-phase keydown handler, a pure `nextNeedsYou` cycle helper (same pattern as `pane-nav.ts`), and one `ApproveButton` component reused by both `TerminalPane` and `Landing` — arm-then-confirm, identical idiom to the existing delete confirmation.

**Tech Stack:** Electron (main/preload/renderer), React 19, xterm.js, node-pty, vitest (unit, `tests/unit/**` only), Playwright `_electron` (e2e).

Spec: `docs/superpowers/specs/2026-07-06-localflow-v2-roadmap.md` § "M2.5 — Needs-you quick actions".

## Global Constraints

- Conventional Commits, subject ≤ 50 chars — enforced by husky commit-msg hook and CI commitlint. Commit body lines ≤ 100 chars.
- `npm run check` (eslint + prettier + typecheck + vitest) must pass before every commit.
- e2e (`npm run e2e`) runs Playwright against the built app; e2e affordances are gated behind `LOCALFLOW_E2E=1`.
- Config-as-code: `keybindings.json` in userData is the source of truth; every new action must be remappable there and validated by `mergeBindings`.
- Every feature serves both audiences: glanceable GUI (Approve buttons) AND keyboard-first (`cmd+u`).
- DOM contract used by CSS + e2e: `.pane[data-pane-id][data-status]`, `.session-row[data-session-id]`, `.dot[data-status]`. Do not rename these.
- Button mousedown discipline: every new button gets `onMouseDown={(e) => e.preventDefault()}`; buttons inside a pane header ALSO call `e.stopPropagation()` (clicking must not activate the pane or steal terminal focus).
- Never blind-approve: the confirm control must always render alongside the peek of recent output.

- [ ] **Step 3: Create `src/main/peek.ts`**

MOVE the `ANSI_RE` declaration (including its full explanatory comment block, `session-manager.ts` lines 68–86) into this new file verbatim, add `export` to the const, and add `extractPeekLines`:

```ts
// Strips ANSI/VT escape sequences per the ECMA-48 grammar: CSI with full
// parameter bytes (covers private-mode like ESC[>0q), OSC titles ended by
// BEL/ST, DCS-family strings, other C1 escapes, and stray control bytes.
// Also covers the 8-bit C1 CSI () form some agents/terminfo emit
// instead of the 7-bit ESC[ prefix — same CSI grammar, single code unit.
// Partial stripping here leaks garbage like "0q4mu" into user messages.

export const ANSI_RE = new RegExp(
  [
    '\\u001b\\[[0-9:;<=>?]*[ -/]*[@-~]',
    '\\u009b[0-9:;<=>?]*[ -/]*[@-~]',
    '\\u001b\\][^\\u0007\\u001b]*(?:\\u0007|\\u001b\\\\)?',
    '\\u001b[PX^_][^\\u001b]*(?:\\u001b\\\\)?',
    '\\u001b[ -/]+[0-~]',
    '\\u001b[@-Z\\\\-_]',
    '[\\u0000-\\u0008\\u000b-\\u001f\\u007f]'
  ].join('|'),
  'g'
)

/**
 * Last `maxLines` non-empty, ANSI-stripped output lines from a raw pty
 * buffer — the "pending question" peek shown next to an Approve control.
 * Best-effort by design: full-screen TUI redraw frames concatenate after
 * stripping, and bare-\r progress updates collapse (\r is a stripped
 * control byte), but the last screenful an agent printed is what the last
 * few lines land on. Splits on \n only — \r never survives the strip.
 */
export function extractPeekLines(raw: string, maxLines: number): string[] {
  return raw
    .replace(ANSI_RE, '')
    .split('\n')
    .map((l) => l.trimEnd())
    .filter((l) => l.length > 0)
    .slice(-maxLines)
}
```

NOTE (copy exactly): the `` in the comment and the `` pattern are the real 8-bit C1 CSI character — copy the block from `session-manager.ts` rather than retyping it.

- [ ] **Step 3: Implement `src/renderer/src/lib/needs-you.ts`**

```ts
import type { SessionInfo } from '../../../shared/types'

/**
 * Picks the pane the jump-to-attention key should land on: the first
 * needs-you session strictly after `activeId` in display order, wrapping —
 * so repeated presses cycle through every waiting pane, and the active pane
 * itself is returned only when it is the sole one waiting. `activeId` null
 * or unknown (e.g. pressed from the home overview) starts the scan from the
 * top of the order. Pure; returns null when nothing needs attention.
 */
export function nextNeedsYou(
  order: string[],
  sessions: SessionInfo[],
  activeId: string | null
): string | null {
  if (order.length === 0) return null
  const waiting = new Set(
    sessions.filter((s) => s.status === 'needs-you').map((s) => s.id)
  )
  if (waiting.size === 0) return null
  const start = activeId === null ? -1 : order.indexOf(activeId)
  for (let i = 1; i <= order.length; i++) {
    const id = order[(start + i) % order.length]
    if (waiting.has(id)) return id
  }
  return null
}
```

- [ ] **Step 1: Create `src/renderer/src/components/ApproveButton.tsx`**

```tsx
import { useEffect, useRef, useState } from 'react'

interface Props {
  sessionId: string
  /** Trigger-button styling, supplied by the host context (pane header vs row). */
  buttonClassName: string
  /** Pane-header buttons must not bubble mousedown into pane activation. */
  stopMouseDown?: boolean
}

/**
 * The needs-you Approve control: arm-then-confirm, never blind. Arming
 * fetches a peek of the session's recent output (the pending question) and
 * shows it beside the confirm button; confirming writes Enter — the
 * agent-agnostic "accept the highlighted option" keystroke — to the
 * session's pty. Outside click or bare Escape disarms, same idiom as the
 * overview rows' delete confirmation.
 */
export default function ApproveButton({
  sessionId,
  buttonClassName,
  stopMouseDown
}: Props): React.JSX.Element {
  // null = disarmed; a string[] (possibly empty) = armed, holding the peek.
  const [peek, setPeek] = useState<string[] | null>(null)
  const wrapRef = useRef<HTMLSpanElement>(null)

  useEffect(() => {
    if (peek === null) return
    const onDocMouseDown = (e: MouseEvent): void => {
      if (!wrapRef.current?.contains(e.target as Node)) setPeek(null)
    }
    const onDocKeyDown = (e: KeyboardEvent): void => {
      if (e.key === 'Escape' && !e.metaKey && !e.ctrlKey && !e.altKey) setPeek(null)
    }
    window.addEventListener('mousedown', onDocMouseDown)
    window.addEventListener('keydown', onDocKeyDown)
    return () => {
      window.removeEventListener('mousedown', onDocMouseDown)
      window.removeEventListener('keydown', onDocKeyDown)
    }
  }, [peek])

  const onMouseDown = (e: React.MouseEvent): void => {
    // preventDefault keeps DOM focus where it is; stopPropagation (pane
    // headers only) keeps the click from activating the pane.
    e.preventDefault()
    if (stopMouseDown) e.stopPropagation()
  }

  const approve = (): void => {
    // \r is what a terminal Enter sends through a pty.
    window.localflow.write(sessionId, '\r')
    setPeek(null)
  }

  return (
    <span className="relative" ref={wrapRef}>
      <button
        className={`approve-btn ${buttonClassName}`}
        onClick={() => void window.localflow.peekSession(sessionId).then(setPeek)}
        onDoubleClick={(e) => e.stopPropagation()}
        onMouseDown={onMouseDown}
      >
        approve
      </button>
      {peek !== null && (
        <div
          className="approve-pop bg-surface-raised absolute top-full right-0 z-20 mt-1 w-72 rounded-md border border-yellow-500/40 p-2 shadow-lg"
          onMouseDown={(e) => {
            // The popover floats over other mousedown-sensitive chrome
            // (terminal, pane root) — interacting with it must not
            // activate/focus anything underneath.
            e.preventDefault()
            if (stopMouseDown) e.stopPropagation()
          }}
        >
          <pre className="approve-peek m-0 mb-2 max-h-28 overflow-auto font-mono text-[10px] leading-relaxed whitespace-pre-wrap text-gray-300">
            {peek.length > 0 ? peek.join('\n') : '(no recent output)'}
          </pre>
          <div className="flex justify-end gap-1.5">
            <button
              className="approve-cancel cursor-pointer rounded-md border border-white/10 bg-white/[0.07] px-2 py-1 text-xs text-gray-300 hover:bg-white/[0.13] hover:text-white"
              onClick={() => setPeek(null)}
              onMouseDown={onMouseDown}
            >
              Cancel
            </button>
            <button
              className="approve-confirm cursor-pointer rounded-md border border-yellow-500/50 bg-yellow-500/15 px-2 py-1 text-xs text-yellow-300 hover:bg-yellow-500/25"
              onClick={approve}
              onMouseDown={onMouseDown}
            >
              Send ⏎
            </button>
          </div>
        </div>
      )}
    </span>
  )
}
```

```ts
test('cmd+u enters terminals on a waiting pane and cycles', async () => {
  const userData = mkdtempSync(join(tmpdir(), 'localflow-e2e-'))
  const app = await launchApp(userData)
  const win = await app.firstWindow()
  await win.setViewportSize({ width: 1400, height: 900 })
  await expect(win.locator('.new-session')).toBeVisible()

  const createSession = (cwd: string): Promise<{ id: string } | null> =>
    win.evaluate(
      (dir) =>
        (
          window as unknown as {
            localflow: { createSession(a: string, c: string): Promise<{ id: string } | null> }
          }
        ).localflow.createSession('claude', dir),
      cwd
    )

  const a = await createSession(userData)
  const b = await createSession(userData)
  const c = await createSession(userData)
  await expect(win.locator(`[data-session-id="${c!.id}"]`)).toBeVisible()

  const { port, token } = JSON.parse(readFileSync(join(userData, 'endpoint.json'), 'utf8'))
  const post = (paneId: string, event: string): Promise<Response> =>
    fetch(`http://127.0.0.1:${port}/event`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json', 'X-Localflow-Token': token },
      body: JSON.stringify({ paneId, event })
    })

  await post(b!.id, 'Notification')
  await post(c!.id, 'Notification')
  const rowB = win.locator(`[data-session-id="${b!.id}"]`)
  await expect(rowB.locator('.session-status')).toHaveText('needs you')

  // From the home overview, cmd+u enters the terminals view directly on the
  // first waiting pane, enlarged (3 sessions exist).
  await win.keyboard.press('Meta+u')
  const paneB = win.locator(`[data-pane-id="${b!.id}"]`)
  const paneC = win.locator(`[data-pane-id="${c!.id}"]`)
  await expect(paneB).toBeVisible()
  await expect(paneB).toHaveClass(/active/)
  await expect(paneB).toHaveClass(/enlarged/)

  // Again: cycles to the next waiting pane.
  await win.keyboard.press('Meta+u')
  await expect(paneC).toHaveClass(/active/)
  await expect(paneC).toHaveClass(/enlarged/)
  await expect(paneB).not.toHaveClass(/enlarged/)

  // Again: wraps back.
  await win.keyboard.press('Meta+u')
  await expect(paneB).toHaveClass(/active/)

  // Nothing waiting -> no-op: focus stays put.
  await post(b!.id, 'Stop')
  await post(c!.id, 'Stop')
  await expect(paneB).toHaveAttribute('data-status', 'idle')
  await expect(paneC).toHaveAttribute('data-status', 'idle')
  await win.keyboard.press('Meta+u')
  await expect(paneB).toHaveClass(/active/)
  await expect(win.locator(`[data-pane-id="${a!.id}"]`)).not.toHaveClass(/active/)

  await app.close()
})
```

## Self-Review Notes

**Spec coverage** (`roadmap § M2.5`):
- "keybinding (default cmd+u) focuses + enlarges the next needs-you pane; press again to cycle" → Task 2 (action + cycle helper) + Task 4 e2e. Enlarge semantics delegated to the existing `openSession` (enlarges when >1 session — matches how row-open behaves everywhere else in the app).
- "on yellow panes (header) and overview rows, an Approve action writes the confirm keystroke (Enter) to that session's pty" → Task 3 (header) + Task 4 (rows); write path reuses the existing `session:write` IPC, no new privileged channel.
- "never blind-approve — show a peek of the pending question beside the approve control" → arm-then-confirm popover; the confirm button only exists inside the popover that contains the peek.
- "requires the per-session output ring buffer in main … its first real consumer" → Task 1 derives the peek from the existing raw `tail` (cap raised 2000 → 16384) instead of adding a second buffer — one buffer, two consumers, DRY. This intentionally interprets "ring buffer" as "bounded recent-output buffer"; a line-oriented ring can replace it later without changing the IPC surface.

**Type consistency:** `peekSession(id, maxLines?) => Promise<string[]>` (Tasks 1/3), `extractPeekLines(raw, maxLines) => string[]` (Task 1), `nextNeedsYou(order, sessions, activeId) => string | null` (Tasks 2/4), `ApproveButton` props `{sessionId, buttonClassName, stopMouseDown?}` (Tasks 3/4) — all uses match.

**Known risks / accepted trade-offs:**
- Peek quality on heavy-TUI agents is best-effort (redraw frames concatenate after ANSI strip). Honest behavior, documented in `extractPeekLines`'s comment.
- The approve `\r` is agent-agnostic "accept the highlighted option"; for prompts whose default is not the confirm choice, the peek is exactly what lets the user decide not to approve.
- `session-manager.test.ts` step gives indicative test code; the implementer must adapt fixture-helper names to the file's actual `setup()`/spec factory idioms.
- cmd+u e2e asserts `enlarged` on cycle targets — relies on `openSession`'s >1-session enlarge rule; if that rule changes, the e2e flags it (intended: they are one semantic).
