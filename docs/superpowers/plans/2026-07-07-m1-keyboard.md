# M1 Keyboard Core Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Active-pane model with cyan focus ring, cmd-hjkl directional focus / cmd-shift-hjkl swap, remappable keybindings.json, and keyboard-focus correctness in the Terminals view.

**Architecture:** A pure shared keybindings module (parse/match/merge) is loaded by main from userData and handed to the renderer read-only over IPC. The renderer owns the active-pane state, a pane-order array, a capture-phase keydown dispatcher, and pure geometry helpers for directional moves. Spec: `docs/superpowers/specs/2026-07-07-m1-keyboard-design.md`.

**Tech Stack:** existing app (Electron, React, TS strict, Tailwind v4 utilities + small plain-CSS design-system file, Vitest, Playwright).

## Global Constraints

- Conventional Commits, subject ≤50 chars (commitlint-enforced).
- TypeScript strict, no `any`. Tailwind for new layout/spacing/colors in TSX; status/data-attribute design-system rules stay in `styles.css`.
- Bare Escape/Enter/unmodified keys must always reach the terminal; only bound combos are intercepted.
- Active ring color: `#00ffff`, as an outer `box-shadow` ring — never touches the status border color.
- Default bindings exactly per the spec table (`focus-left: cmd+h` … `go-up: cmd+escape`).
- E2E DOM contract additions: active pane root gains class `active`; everything existing (`data-pane-id`, `data-status`, `.new-session`, `.row-open`, `data-session-id`) unchanged.
- Existing behavior that must not regress: cmd+esc = shrink-if-enlarged-else-home; session create → Terminals view.

### Task 1: Shared keybindings module (TDD)

**Files:**
- Create: `src/shared/keybindings.ts`
- Test: `tests/unit/keybindings.test.ts`

**Interfaces (produces — later tasks import these exact names):**

```ts
export type KeyAction =
  | 'focus-left' | 'focus-down' | 'focus-up' | 'focus-right'
  | 'swap-left' | 'swap-down' | 'swap-up' | 'swap-right'
  | 'enlarge-toggle' | 'close-pane' | 'new-session' | 'go-up'

export const DEFAULT_BINDINGS: Record<KeyAction, string>
// { 'focus-left': 'cmd+h', 'focus-down': 'cmd+j', 'focus-up': 'cmd+k',
//   'focus-right': 'cmd+l', 'swap-left': 'cmd+shift+h', 'swap-down': 'cmd+shift+j',
//   'swap-up': 'cmd+shift+k', 'swap-right': 'cmd+shift+l',
//   'enlarge-toggle': 'cmd+m', 'close-pane': 'cmd+w',
//   'new-session': 'cmd+enter', 'go-up': 'cmd+escape' }

export interface ParsedBinding { cmd: boolean; ctrl: boolean; alt: boolean; shift: boolean; key: string }
export function parseBinding(binding: string): ParsedBinding | null
export interface KeyEventLike { key: string; metaKey: boolean; ctrlKey: boolean; altKey: boolean; shiftKey: boolean }
export function eventMatches(binding: ParsedBinding, e: KeyEventLike): boolean
export function mergeBindings(user: unknown): Record<KeyAction, string>
```

Semantics:
- `parseBinding`: grammar `[cmd+][ctrl+][alt+][shift+]<key>`, case-insensitive, modifiers in any order; `<key>` = single char or one of `enter|escape|tab|space|arrow-left|arrow-right|arrow-up|arrow-down`. Returns null for empty key, unknown named key, duplicate modifiers, or no `+`-separated structure.
- `eventMatches`: modifiers must match exactly (cmd↔metaKey, etc.); key comparison case-insensitive; named keys map to KeyboardEvent.key values (`Enter`, `Escape`, `Tab`, ` `, `ArrowLeft`, …).
- `mergeBindings(user)`: start from `DEFAULT_BINDINGS`; if `user` is an object, override only entries whose action name is a known `KeyAction` AND whose value parses via `parseBinding`. Anything else silently ignored.

### Task 4: Keyboard dispatcher + directional nav + swap (TDD for geometry)

**Files:**
- Create: `src/renderer/src/lib/pane-nav.ts`
- Modify: `src/renderer/src/App.tsx`
- Test: `tests/unit/pane-nav.test.ts`

**Interfaces:**
- Produces: `interface PaneRect { id: string; x: number; y: number; w: number; h: number }`; `pickNeighbor(rects: PaneRect[], activeId: string, dir: 'left'|'down'|'up'|'right'): string | null` — from the active rect's center, candidates are rects whose center lies strictly in the given direction (dx<0 for left etc., using the dominant axis: for left/right require `|dx| > |dy|` is NOT required — instead score = primary distance + 2×orthogonal offset, pick lowest score; no candidates → null); `swapInOrder(order: string[], a: string, b: string): string[]` (pure, returns new array).
- Consumes: Task 1 module (`parseBinding`, `eventMatches`, `mergeBindings` result via `getKeybindings`), Task 3 state.

- [ ] **Step 1 (TDD):** failing tests for `pickNeighbor` on a 2×2 grid layout (four 100×100 rects at (0,0),(110,0),(0,110),(110,110)): right/left/up/down from each corner, null at edges; ties broken by orthogonal proximity (add a third column offset vertically and assert the aligned one wins). Tests for `swapInOrder` (adjacent, non-adjacent, unknown id → unchanged).
- [ ] **Step 2:** RED recorded. Implement. GREEN.
- [ ] **Step 3:** App wiring: on mount fetch `getKeybindings()`, parse once into `[action, ParsedBinding][]`. Replace the existing cmd+esc-specific handler with a capture-phase dispatcher `window.addEventListener('keydown', handler, true)`:
  - find first action whose binding `eventMatches` the event; if none → return (terminal gets the key).
  - `e.preventDefault(); e.stopPropagation()`.
  - Actions: `go-up` → existing shrink-else-home logic. `new-session` → `setView('home')`. Others only in Terminals view with an active pane: `enlarge-toggle` → toggle enlarged(active); `close-pane` → `close(active)`; `focus-*` → `pickNeighbor` over rects harvested via `document.querySelectorAll('.pane')` → `getBoundingClientRect()` mapped to `PaneRect` (skip when enlarged — focus moves are a no-op while a pane is enlarged); `swap-*` → pickNeighbor then `setOrder(swapInOrder(order, active, neighbor))` keeping active unchanged.
